# 사내용 AS 시스템 개발 명세서

## 1. 프로젝트 개요

- **프로젝트명:** 사내용 AS 관리 시스템 (Internal AS System)
- **목표:** AS 입고부터 수리, 출고, 이력 관리까지의 전 과정을 전산화하여 업무 효율성 증대 및 데이터 누락 방지.
- **핵심 가치:** 쉽고 빠른 입력(드롭다운, 일괄 입고), 정확한 이력 추적, 마스터 데이터 기반의 체계적 관리.

---

## 2. 기술 스택 (Tech Stack)

| 구분            | 기술              | 설명                                                            |
| :-------------- | :---------------- | :-------------------------------------------------------------- |
| **Language**    | **Python**        | 전체 비즈니스 로직 처리 및 백엔드 구현                          |
| **Framework**   | **Django**        | 웹 애플리케이션 프레임워크 (MVT 패턴, ORM 활용)                 |
| **Database**    | **SQLite** (초기) | 개발 및 초기 단계용 내장 DB (추후 PostgreSQL 마이그레이션 고려) |

| **Editor**      | **VS Code**       | 통합 개발 환경                                                  |

## 3. 기능 구현 현황 (Implementation Status)

### 3.1. 마스터 데이터 관리 ✅ 구현 완료
- **업체 (Company):** 업체명, 업체 구분(매출처/의뢰처), 지역/국가, 주소 관리.
- **단가 그룹 (CompanyCategory):** 업체별 단가 적용을 위한 그룹 관리 (업체 관리에서 통합).
- **브랜드 (Brand) & 툴 (Tool):** 브랜드별 장비 모델 관리. 브랜드 상세에서 툴 인라인 편집 가능.
- **부품 (Part):** 부품명, 코드, 단가, 구분(전용/공용/공임) 관리. `filter_horizontal`로 여러 툴에 적용 가능.
- **의뢰업체 (OutsourceCompany):** 외주 수리 업체 관리.

### 3.2. AS 프로세스 ✅ 구현 완료
- **일괄 입고 (InboundBatch):**
    - 날짜, 업체, 담당자 등 공통 정보를 1번 입력.
    - `TabularInline`을 통해 여러 장비(시리얼 포함)를 한 번에 등록.
    - 쉼표(`,`)로 구분된 시리얼 번호 입력 시 자동으로 개별 티켓 생성.
    - 중복 입고 방지 로직 적용.
- **AS 티켓 (ASTicket):**
    - 입고~출고 전 과정을 하나의 티켓으로 관리.
    - 상태: 입고(Inbound) → 수리대기(Waiting) → 수리완료(Repaired) / 수리의뢰(Outsourced) / 자체폐기(Disposed) → 출고(Shipped).
- **수리 기록 (RepairTicket):**
    - 입고 상태인 티켓만 필터링하여 표시.
    - 부품/공임(M2M) 선택 시 수리 비용 자동 계산.
    - 외주 수리 시 의뢰업체 지정 가능.
- **출고 처리 (OutboundTicket):**
    - 수리 완료된 티켓만 필터링.
    - 목록에서 체크박스 선택 후 '출고 처리' 액션으로 일괄 상태 변경.
    - 견적서 추출 여부, 세금계산서 발행 여부 체크.
- **통합 이력 (ASHistory):**
    - 전체 이력 조회 전용 (Read-Only).
    - 다양한 필터(업체, 날짜, 상태 등) 및 검색 제공.

## 4. 데이터베이스 설계 (Schema Design)

### 4.1. 주요 모델
- **CompanyCategory:** 단가 그룹
- **Company:** 업체 정보 (단가 그룹 FK 포함)
- **Brand:** 브랜드 정보
- **Tool:** 장비 정보 (브랜드 FK)
- **Part:** 부품 정보 (Tool M2M - 다중 장비 호환)
- **OutsourceCompany:** 외주 업체 정보

### 4.2. 트랜잭션 모델
- **InboundBatch:** 일괄 입고 헤더 (날짜, 업체, 담당자)
- **ASTicket:** AS 통합 티켓
    - FK: InboundBatch, Company, Tool, OutsourceCompany
    - M2M: Used Parts (사용 부품)
    - Status: 입고/대기/완료/의뢰/폐기/출고
    - Dates: 입고일, 출고일
    - Flags: 견적서 발행, 세금계산서 발행

## 5. UI/UX 가이드라인 (Django Unfold)

현재 시스템은 **Django Unfold** 테마를 기반으로 하여, 직관적이고 모던한 Admin UI를 제공합니다.

### 5.1. 일반 원칙
- **Clean & Minimal:** 불필요한 장식 요소를 배제하고 데이터 가독성을 최우선으로 합니다.
- **No Popup:** 팝업 창을 띄우는 대신, 가능한 한 현재 페이지 내에서 인라인 편집이나 자동완성을 활용합니다.
- **Keyboard Friendly:** 탭(Tab) 키 이동과 엔터(Enter) 키 제출이 자연스럽게 이어지도록 필드 순서를 배치합니다.

### 5.2. 입력 화면 (Forms)
- **자동완성 (Autocomplete):**
    - 업체(Company), 장비(Tool) 등 데이터가 많은 필드는 반드시 `autocomplete_fields`를 사용하여 검색 가능하도록 함.
    - 브라우저 기본 자동완성(`autocomplete="off"`)은 비활성화하여 데이터 입력을 방해하지 않음.
- **인라인 편집 (Inlines):**
    - 입고 등록 시 `TabularInline`을 사용하여 엑셀처럼 빠르게 여러 행을 입력할 수 있도록 구성.
    - 불필요한 추가/수정/삭제 아이콘(`NoRelatedButtonsMixin`)을 제거하여 입력 실수 방지.
- **필드 그룹화 (Fieldsets):**
    - 관련 있는 필드끼리 묶어서 표시 (예: '수리 정보', '출고 및 정산').
    - 중요도가 낮은 필드(메모 등)는 하단이나 별도 영역으로 배치.

### 5.3. 목록 화면 (ChangeList)
- **필터링 (Filters):**
    - 우측 사이드바 필터를 활용하여 상태(Status), 업체(Company), 날짜(Date)별 좁혀보기 기능 제공.
- **검색 (Search):**
    - 시리얼 번호, 업체명, 모델명으로 통합 검색 지원.
- **일괄 액션 (Actions):**
    - 여러 항목을 선택하여 상태를 한 번에 변경하는 기능 제공 (예: '선택된 항목 수리완료 처리').
- **상태 표시:**
    - 상태(Status) 뱃지나 색상을 활용하여 진행 상황을 시각적으로 구분 (Unfold 기본 스타일 활용).

### 5.4. 네비게이션 (Navigation)
- **사이드바 구성:**
    - **AS 관리:** 업무 흐름 순서대로 배치 (입고 → 수리 → 출고 → 이력).
    - **자재/기준정보:** 관리자만 접근하거나 빈도가 낮은 메뉴는 별도 섹션으로 분리.
    - 사용하지 않는 메뉴(예: 개별 입고 티켓 목록)는 사이드바에서 숨김 처리 (`has_module_permission = False`).

## 6. 향후 개선 계획 (Roadmap)

- **견적서 출력:** 엑셀 폼 연동하여 견적서 파일 생성 및 다운로드 기능.
- **대시보드 강화:** 메인 화면에 금일 입고/출고 건수, 수리 대기 건수 등 KPI 카드 추가.
- **모바일 대응:** 현장에서 모바일 기기로 사진 첨부 및 간단한 상태 변경 가능하도록 반응형 최적화.
