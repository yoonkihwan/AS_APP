# 사내용 AS 시스템 개발 명세서

## 1. 프로젝트 개요

- **프로젝트명:** 사내용 AS 관리 시스템 (Internal AS System)
- **목표:** AS 입고부터 수리, 출고, 이력 관리까지의 전 과정을 전산화하여 업무 효율성 증대 및 데이터 누락 방지.
- **핵심 가치:** 쉽고 빠른 입력(드롭다운, 일괄 입고), 정확한 이력 추적, 마스터 데이터 기반의 체계적 관리.
- **포탈:** 통합 포탈 페이지(`/`)에서 로그인, 각 서브시스템 바로가기, 개선사항 요청 게시판을 제공.

---

## 2. 기술 스택 (Tech Stack)

| 구분            | 기술              | 설명                                                            |
| :-------------- | :---------------- | :-------------------------------------------------------------- |
| **Language**    | **Python**        | 전체 비즈니스 로직 처리 및 백엔드 구현                          |
| **Framework**   | **Django 5.x**    | 웹 애플리케이션 프레임워크 (MVT 패턴, ORM 활용)                 |

| **Import/Export** | **django-import-export** | 데이터 일괄 가져오기/내보내기 지원                        |

## 3. 앱 구성 (App Architecture)

| Django 앱        | 경로 접두사     | 역할                                                         |
|:----------------|:---------------|:------------------------------------------------------------|
| **master_data** | `/master/`     | 공용 기준정보 (Company, Brand, Tool, OutsourceCompany 등) — 모든 앱이 공유하는 Single Source of Truth |
| **as_app**      | `/admin/`      | AS 접수·수리·출고·이력·견적서·세금계산서 프로세스 전체          |
| **tool_inventory** | `/inventory/` | 장비/툴 입고·출고·재고 현황 관리                               |
| **hr_app**      | `/hr/`         | 근무/근태 관리 (달력 UI, 근태 기록)                             |
| **as_project/sysadmin** | `/sysadmin/` | 시스템 계정 관리 및 활동 모니터링                          |

## 4. 기능 구현 현황 (Implementation Status)

### 4.1. 마스터 데이터 관리 ✅ 구현 완료
- **공용 기준정보 앱 분리 (`master_data`):**
    - `Company`, `CompanyCategory`, `Brand`, `Tool`, `OutsourceCompany` 모델을 별도 앱으로 분리.
    - 기존 DB 테이블명(`as_app_*`)을 유지하여 무중단 마이그레이션 달성 (`db_table` 명시).
    - `as_app`에서는 `from master_data.models import ...`으로 re-export하여 기존 코드 하위 호환성 100% 유지.
- **매출처 (Company):** 업체명, 사업자등록번호, 대표자, 주소, 단가 그룹 관리.
- **단가 그룹 (CompanyCategory):** 업체별 단가 적용을 위한 그룹 관리. 기준정보 포탈(`/master/`)에서 탭 UI로 통합 관리.
- **브랜드 (Brand) & 툴 (Tool):** 브랜드별 장비 모델 관리. AS 시스템과 Tool Inventory가 동일 모델을 공유.
- **부품 (Part):** 부품명, 코드, 구분(`part_type`: 부품/공임) 관리. `filter_horizontal`로 여러 툴에 적용 가능. 부품 목록에서 구분 필터를 통해 부품과 공임을 분류 조회 가능.
- **부품 단가 (PartPrice):** 단가 그룹별 차등 단가 관리. 부품 편집 폼에서 각 단가 그룹의 가격을 직접 입력 (동적 폼 필드 생성). "다스" 그룹이 기본 단가 기준.
- **수리 세트 (RepairPreset):** 자주 사용하는 부품 조합을 프리셋으로 관리. 브랜드 선택 시 해당 브랜드 장비만 Cascading Dropdown으로 필터링.
- **의뢰업체 (OutsourceCompany):** 외주 수리 업체 관리. 기준정보 포탈(`/master/`)에서 통합 관리.
- **기준정보 포탈 (`/master/`):** 독립 AdminSite로 매출처, 의뢰업체, 단가그룹, 브랜드&툴을 탭 네비게이션으로 통합 관리.

### 4.2. 업체관리 ✅ 구현 완료
- AS 관리 사이드바의 **"업체관리"** 메뉴에서 매출처 관리.
- 기준정보 포탈(`/master/`)에서 매출처, 의뢰업체, 단가그룹을 **탭 네비게이션**으로 전환 가능.

### 4.3. AS 프로세스 ✅ 구현 완료
- **일괄 입고 (InboundBatch):**
    - 날짜, 업체, 담당자 등 공통 정보를 1번 입력.
    - `TabularInline`을 통해 여러 장비(시리얼 포함)를 한 번에 등록.
    - 브랜드 선택 → 해당 브랜드 장비만 필터링 (Cascading Dropdown).
    - 쉼표(`,`)로 구분된 시리얼 번호 입력 시 자동으로 개별 티켓 생성.
    - **시리얼 없음 체크박스 + 수량 입력:** 시리얼 번호가 없는 장비는 체크 후 수량 입력 시 `없음-{UUID6자리}` 형태로 자동 생성.
    - 동일 배치 내 중복 검증 + 기존 활성 티켓 중복 검증.
    - 저장 후 자동으로 새 입고 등록 폼으로 리다이렉트 (연속 입고 UX).
- **AS 티켓 (ASTicket):**
    - 입고~출고 전 과정을 하나의 티켓으로 관리.
    - 상태: 입고(Inbound) → 수리의뢰(Outsourced) / 수리완료(Repaired) / 자체폐기(Disposed) → 출고(Shipped).
    - **의뢰 정보 필드:** 의뢰업체(FK), 의뢰 날짜를 티켓에 직접 기록.
- **수리 의뢰 등록 (OutsourcedTicket):** ✅ 신규
    - AS 사이드바의 **"수리 의뢰 등록"** 메뉴로 독립 화면 제공.
    - 입고 상태인 티켓만 목록에 표시.
    - 목록에서 **개별 "📦 수리 의뢰 등록" 버튼** 클릭 시 의뢰업체 선택 + 날짜 입력 중간 페이지 표시.
    - 체크박스 선택 후 **"📦 선택 항목 수리의뢰 처리"** 일괄 액션 지원.
    - 의뢰 처리 시 비고(repair_content)에 `{업체명} 수리의뢰 ({날짜}~)` 자동 기록.
- **수리 기록 (RepairTicket):**
    - 입고/수리의뢰 상태인 티켓만 필터링하여 표시.
    - 목록에서 **개별 "🔧 수리 기록 등록" 버튼**으로 수리 등록 페이지 이동 (행 클릭 비활성화).
    - 해당 장비에 적용 가능한 부품만 **커스텀 테이블 위젯(`PartsTableWidget`)**으로 표시 (부품/공임 섹션 시각적 분리, 부품코드·단가 표시).
    - **업체 단가 그룹 미설정 시 저장 차단:** 단가 그룹이 없는 업체의 티켓은 경고 메시지와 함께 수리 기록 저장이 불가. 업체관리 링크 제공.
    - **업체별 차등 단가 자동 적용:** 수리 부품 선택 시 해당 업체의 단가 그룹을 인식하여 차등 단가가 자동 적용됨.
    - **수리 비용 자동 계산:** 스냅샷 단가 합산 × 0.9(네고가) × 1.1(부가세) = 최종 AS 비용. `TicketUsedPart` 모델을 통해 수리 시점 단가를 스냅샷으로 보존.
    - 부품/공임(M2M) 선택 시 수리완료 상태 자동 변경.
    - **수리의뢰 → 수리완료 전환 시:** 비고에 기록된 의뢰 기간(`~)`)을 자동으로 완료일(`~{오늘}`)로 업데이트.
    - **수리 세트 프리셋 (다중 선택):** 여러 수리 세트를 토글 방식으로 동시 적용 가능. 부품은 누적 선택되며, **공임은 선택된 세트들 중 최고가 공임 1개만 자동 선택** (중복 과금 방지).
    - 부품 목록 API (`api/parts-for-tool/<tool_id>/`) 제공.
- **출고 처리 (OutboundTicket):**
    - 수리 완료된 티켓만 필터링.
    - 목록에서 **개별 "📤 출고 등록" 버튼** 클릭 시 출고 날짜 선택 중간 페이지 표시.
    - 체크박스 선택 후 **오늘 날짜 출고 / 날짜 선택 출고** 두 가지 일괄 액션 제공.
    - 목록에 **수리 내역 아코디언** 표시 (공임 먼저, 가격순 정렬, 부품코드·단가 포함).
- **통합 이력 (ASHistory):**
    - 전체 이력 상세 조회 및 **제한적 수정 가능** (마스터 데이터인 매출처, 장비/툴은 `autocomplete_fields`로 검색/선택만 가능, 임의 입력 불가).
    - **사용 부품 인라인 편집:** `TicketUsedPartInline`을 통해 수리 사용 부품/공임 스냅샷을 인라인으로 조회·수정 가능.
    - **복합 컬럼 UI:** 입고일/출고일, 매출처·담당자, 장비·S/N, 비용·증빙(견적✓✗/세금✓✗) 등 관련 정보를 2줄 셀로 압축 표시.
    - **수리 내역 아코디언:** 사용 부품/공임을 `<details>` 태그로 접기/펼치기 (공임먼저 → 가격순 정렬).
    - **상태 정정 액션:** 체크박스 선택 후 상태 변경 가능 (혼합 선택 시 에러 메시지 표시).
        - 출고 취소 → 수리완료로 되돌리기 (출고일 초기화)
        - 수리완료 취소 → 입고로 되돌리기 (사용 부품/공임 초기화, 수리 비용 0원)
        - 수리의뢰 취소 → 입고로 되돌리기 (의뢰업체/날짜 초기화)
        - 자체폐기 취소 → 입고로 되돌리기
        - 견적서 추출 상태 초기화
        - 세금계산서 발행 상태 초기화
        - 입고 → 자체폐기 처리
    - **삭제 기능:** 목록에서 체크박스 선택 후 삭제 액션 지원 (Django 기본 삭제 확인 화면 표시).
    - **고급 필터:** 상태, 매출처, 브랜드, 입고일(기간), 출고일(기간) 다중 필터 + 시리얼/업체/모델/증상/비고 검색.
- **견적서 발행 (추출 메뉴):**
    - AS 관리 사이드바의 '추출' 그룹으로 분리.
    - 수리완료 또는 출고 상태인 데이터들을 대상으로 **미리보기 및 수정 페이지** 제공.
    - 미리보기에서 업체명, 모델명, 부품 정보, 단가, 수량, 비고를 확인/수정 후 PDF 다운로드.
    - **업체별 단가 그룹 자동 적용:** 견적서 생성 시 `TicketUsedPart.applied_price` (스냅샷 단가) 기준으로 네고가(90%) 자동 계산.
    - PDF 다운로드 시 자동으로 `estimate_status=True` 마킹.
- **세금계산서 등록 (AS 관리 메뉴):**
    - 출고 등록 하단에 배치.
    - 출고된 데이터를 조회하여 목록에서 직접 `list_editable`로 '세금계산서 발행' 여부 토글 (내부 확인용).
    - 복합 컬럼(입고일/출고일, 매출처·담당자, 장비·S/N) 적용으로 간결한 화면 구성.
    - 다양한 필터(업체, 날짜 기간, 상태, 브랜드, 세금계산서 여부) 및 검색 제공.
    - **상태별 행 색상 적용:** 목록 화면에서 행(Row) 전체의 배경색을 변경하여 상태를 직관적으로 구별.

### 4.4. 대시보드 ✅ 구현 완료
- **KPI 카드 (클릭 가능):** 각 카드 클릭 시 해당 필터가 적용된 통합 이력 화면으로 이동.
    - 장기 미처리 대기 (30일 이상 경과, 빨간색)
    - 수리 대기 (입고 상태, 노란색)
    - 수리의뢰 중 (외주 의뢰, 황색)
    - 수리 완료/미출고 (출고 대기, 초록색)
- **수리 매출 요약:** 이번 주 / 이번 달 / 이번 년도 수리매출 (출고 완료 건 기준).
- **수리 대기 목록:** 최근 10건 표시 (장비/시리얼 포함).
- **최근 출고 목록:** 최근 10건 표시.

### 4.5. 장비/툴 입출고 관리 (Tool Inventory) ✅ 구현 완료
- **초점:** 기존 파이썬 데스크탑 형태의 재고 관리 프로그램을 웹으로 이관.
- **기능:**
    - AS 시스템과 사이트 분리 (`/inventory/`)
    - **공유 데이터:** `Brand`, `Tool` 마스터 데이터는 `master_data` 앱에 정의되어 AS 앱과 공유 (Single Source of Truth).
    - **통합 입출고 이력:**
        - 상태, 입고일자, 입고처, 출고일자, 출고처, 품목명, 시리얼번호, 적용공정/용도, 수정의 순서로 컬럼이 배치됨.
        - **동적 정렬 기능 지원:** 사용자가 '출고' 상태 필터나 '출고처' 필터를 활성화하면 출고일자 기준으로, '입고처' 필터를 활성화하면 입고일자 기준으로 자동으로 최신순 정렬됨.
    - **입고 등록:** 배치(Batch) 단위 등록 지원, 시리얼 번호 다중 입력시 자동 분리 등록 기능. 입고업체는 `OutsourceCompany` 마스터 데이터 참조.
    - **출고 등록 (인라인 UI 도입):** 
        - 1회 출고 시 여러 장비를 인라인 폼(TabularInline)으로 한 번에 등록.
        - 모델(Tool) 선택 시 비동기(API)로 해당 모델의 **현재 재고량** 실시간 박스 표기 및 **선택 가능한 재고목록(시리얼)**이 체크박스로 노출.
        - 사용자 선택 사항으로 **적용공정/용도**를 입력하면, 선택된 개별 장비(시리얼)들의 통합 이력 데이터에도 자동으로 용도 데이터가 복사됨.
        - 미선택 시 단순 수량 설정 통한 선입선출 출고 로직 지원.
        - 출고업체는 `Company`(매출처) 마스터 데이터 참조.
    - **재고 현황 출력 (ToolStockSummary):** 실시간 툴별 요약 재고와 시리얼 정보들을 제공하며, "재고 내역 출력" 버튼(글로벌 액션) 하나로 전체 데이터를 ReportLab을 활용해 PDF 및 Excel로 깔끔하게 떨어트리는 기능을 지원합니다.
    - **대시보드:** Tool Inventory 전용 대시보드 (`/inventory/`) 제공.
    - **투두리스트 (TodoItem):** 대시보드에 할 일 목록 기능 탑재.

### 4.6. 시스템 계정 관리 (Sysadmin) ✅ 구현 완료
- **초점:** 직원 계정 생성 후 관리자가 권한을 쉽게 부여할 수 있도록 함.
- **기능:** 
    - 시스템 관리 사이트 분리 (`/sysadmin/`) — **관리자(superuser)만 접근 가능**.
    - **회원가입 (`/signup/`):** 별도 회원가입 페이지 제공. 가입 시 `is_active=False`로 설정되어 관리자 승인 전까지 로그인 불가.
    - **역할 체계:** 관리자(🛡️ superuser) / 작업자(🔧 is_staff+is_active) / 미승인(⏳ is_active=False) 시각적 뱃지 표시.
    - **일괄 액션:** "선택 사용자: 작업자로 승인" / "선택 사용자: 관리자로 승격" 액션 제공.
    - **인증 백엔드 (`StaffFullAccessBackend`):** 작업자(is_staff) 이상이면 전체 업무 권한이 자동 부여되는 커스텀 인증 백엔드.
    - **활동 모니터링:** 
        - 개별 사용자 수정 화면에서 `LogEntryInline`으로 해당 사용자의 최근 20건 활동 기록 인라인 표시.
        - **"전체 활동 기록"** 메뉴로 전 시스템 활동 로그 조회 (읽기 전용, 날짜 계층 탐색, 사용자/작업유형/대상 필터).

### 4.7. 근무/근태 관리 (HR App) ✅ 구현 완료
- **초점:** 엑셀로 관리하던 직원들의 연장근무, 휴가, 특근 내역을 웹 상에서 쉽게 기록하고 조회 가능하도록 함.
- **기능:**
    - 근무/근태 관리 사이트 분리 (`/hr/`)
    - **달력(Calendar) UI:** FullCalendar 기반의 직관적인 월별 달력 대시보드 제공 (`/hr/` 접속 시 바로 표시).
    - **근무 유형:** 정상근무, 연장근무, 주말특근, 연차(종일/오전반차/오후반차), 병가, 공가 등 다양한 근무 형태 지원.
    - **이벤트 API (`/hr/api/calendar/`):** FullCalendar가 비동기로 이벤트를 가져오는 JSON API. 일반 직원은 본인 기록만, 관리자(superuser)는 전 직원 기록 조회.
    - **근태 기록 목록:** Unfold Admin 기반 CRUD 목록 (날짜 계층 탐색, 사용자/근무형태 필터).
    - **AttendanceRecord 모델:** 직원(FK), 근무일자, 근무형태, 시작/종료 시간, 연장/특근 시간, 메모. `unique_together=(user, date)`.

### 4.8. 포탈 및 개선사항 게시판 ✅ 구현 완료
- **통합 포탈 (`/`):** 최상위 도메인 접속 시 로그인 통합 + 서브시스템 바로가기 카드 + 개선사항 게시판을 한 화면에 제공.
- **개선사항 요청 게시판 (ImprovementRequest):**
    - 직원이 시스템에 대한 요청/개선사항을 글로 작성.
    - 상태 관리: 접수(PENDING) → 검토 중(IN_PROGRESS) → 반영 완료(COMPLETED) / 반려(REJECTED).
    - 작성자 자동 설정 (`request.user`), 관리자만 상태/답변 변경 가능.
    - 포탈 메인 화면에서 최근 10건 표시 + 인라인 글쓰기 + 삭제(본인/관리자) + 완료 처리(관리자).
    - Admin에서는 사이드바 숨김 처리 (포탈에서 직접 관리).

## 5. 데이터베이스 설계 (Schema Design)

### 5.1. 주요 모델 (마스터 데이터 — `master_data` 앱)
- **CompanyCategory:** 단가 그룹 (`db_table: as_app_companycategory`)
- **Company:** 매출처 정보 (단가 그룹 FK 포함) (`db_table: as_app_company`)
- **Brand:** 브랜드 정보 (AS 시스템과 Tool Inventory 공유) (`db_table: as_app_brand`)
- **Tool:** 장비 정보 (브랜드 FK, unique_together: brand + model_name) (`db_table: as_app_tool`)
- **OutsourceCompany:** 의뢰업체 정보 (업체명, 연락처, 메모) (`db_table: as_app_outsourcecompany`)

### 5.2. AS 전용 모델 (`as_app`)
- **Part:** 부품/공임 정보 (Brand FK, Tool M2M, `part_type`: 부품/공임 구분). `simple_history` 변경 추적 적용.
- **PartPrice:** 단가 그룹별 차등 단가 (Part FK, CompanyCategory FK, unique_together: part + category)
- **TicketUsedPart:** AS 티켓과 사용 부품의 연결 테이블 (수리 시점 단가 스냅샷 보존, `applied_price` 필드)
- **RepairPreset:** 수리 세트 (Part M2M, Tool M2M, Brand FK)
- **InboundBatch:** 입고 배치 헤더 (입고일, 매출처, 담당자)
- **ASTicket:** AS 통합 티켓 (입고~수리~출고 전 과정). 의뢰 정보(`outsource_company` FK, `outsource_date`) 포함.
- **ImprovementRequest:** 개선사항 요청 게시판 (제목, 내용, 작성자, 상태, 관리자 답변)

### 5.3. Tool Inventory 모델 (`tool_inventory` 앱)
- **InventoryBatch:** 장비 입고 헤더 (입고일, 입고처=OutsourceCompany FK)
- **Inventory:** 통합 입출고 이력 / 개별 단품(시리얼 등) 재고 상태 내역 (재고/출고). `usage_process`를 통한 현장 적용 공정 기록 포함. UUID 기반 PK.
- **OutboundBatch:** 장비 출고 헤더 (출고일, 출고처=Company FK)
- **OutboundTicket:** 출고할 대상 품목 목록 (Tool FK, 수량, Inventory M2M, `usage_process`)
- **TodoItem:** 대시보드 투두리스트 항목

### 5.4. 근무/근태 관리 모델 (`hr_app`)
- **AttendanceRecord:** 개별 직원의 일별 근무형태, 시작/종료 시간, 연장/특근 시간, 메모 등 기록. `unique_together=(user, date)`.

### 5.5. 프록시 모델 (Admin 뷰 분리)

## 6. UI/UX 가이드라인 (Django Unfold)

현재 시스템은 **Django Unfold** 테마를 기반으로 하여, 직관적이고 모던한 Admin UI를 제공합니다.

### 6.1. 일반 원칙
- **Clean & Minimal:** 불필요한 장식 요소를 배제하고 데이터 가독성을 최우선으로 합니다. 다크 모드가 기본 테마입니다.
- **No Popup:** 팝업 창을 띄우는 대신, 현재 페이지 내에서 인라인 편집이나 자동완성을 활용합니다.
- **No Related Buttons:** `NoRelatedButtonsMixin`으로 FK/M2M 필드 옆 추가/수정/삭제 버튼을 제거하여 입력 실수 방지.
- **Keyboard Friendly:** 탭(Tab) 키 이동과 엔터(Enter) 키 제출이 자연스럽게 이어지도록 필드 순서를 배치합니다.
- **네비게이션 직관성:** 사이드바 메뉴명과 화면명 완벽 동기화. `CustomTitleMixin`으로 브라우저 탭 및 화면 제목을 통일.
- **시각적 구별성:** 티켓 상태 표시(입고-노란색, 수리완료-파란색, 출고-초록색, 자체폐기-빨간색)에 **행 단위 배경색(Row Colors)** 을 도입하여 분류 식별성 극대화. `StatusColorMixin` + CSS로 구현.
- **로컬라이징 완벽 지원:** '검색', '필터' 등 플레이스홀더를 사용자에 친화적인 한국어로 교체.
- **복합 컬럼 (Multi-line Cell):** `CompositeDisplayMixin`으로 관련 정보를 2줄(main/sub) 셀로 압축 표시 (매출처·담당자, 장비·S/N, 입고일/출고일, 비용·증빙 등).
- **아코디언 UI:** 적용 장비, 수리 내역 등 긴 목록을 `<details>/<summary>` HTML 태그로 접기/펼치기 처리.

### 6.2. 입력 화면 (Forms)
- **자동완성 (Autocomplete):**
    - 업체(Company), 장비(Tool) 등 데이터가 많은 필드는 반드시 `autocomplete_fields`를 사용.
    - 브라우저 기본 자동완성(`autocomplete="off"`)은 비활성화.
- **인라인 편집 (Inlines):**
    - 입고/출고 처리 시 `TabularInline`을 사용하여 빠르게 여러 행을 한 화면에 다중 처리.
- **동적 필드 제어 (Vanilla JS):**
    - 브랜드 선택 → 툴 목록 갱신, 출고 폼 재고 목록/수량 갱신 등 실시간 통신은 순수 JavaScript `fetch` API 기반으로 구성.
- **커스텀 위젯:**
    - 수리 기록 부품 선택: `PartsTableWidget` — 테이블 형태로 부품코드·단가·체크박스 배치.
    - 출고 대상 시리얼 선택: `CheckboxSelectMultiple` 기반 다중 선택.
- **필드 그룹화 (Fieldsets):**
    - 관련 있는 필드끼리 묶어서 표시 (예: '입고 정보', '수리 부품 선택', '의뢰 정보', '출고 및 정산').
- **저장 후 동작:**
    - 불필요한 부가 저장 버튼(저장 후 계속 편집, 저장 후 새로 추가 등)은 일괄 숨김 처리.
- **개별 액션 버튼:** 출고 등록, 수리 기록, 수리 의뢰 등록 화면에서 각 행에 개별 처리 버튼(📤/🔧/📦) 제공.

### 6.3. 목록 화면 (ChangeList)
- **일괄 액션 (Actions):** 체크박스 선택 후 일괄 출고 처리, 수리의뢰 처리, 상태 정정, 권한 부여 등의 효율적인 액션 활용.
- **탭 네비게이션:** 기준정보 포탈 등에서 상단 탭을 통해 데이터 성격을 분류 (매출처/의뢰업체/단가그룹/브랜드/툴/부품/수리세트).
- **기간 필터 (RangeDateFilter):** Unfold 확장 필터로 입고일/출고일 기간 범위 검색 지원.

### 6.4. 다중 사이트 네비게이션 (Navigation)
하나의 프로젝트에 5개의 독립적인 하위 사이트(AdminSite)를 분리운영하며, 각 사이트별 사이드바/제목/아이콘이 동적으로 변경됩니다.
- **AS 관리 (`/admin/`):** 
    - 대시보드 · 통합 이력 · 입고 등록 · 수리 의뢰 등록 · 수리 기록 등록 · 출고 등록 · 세금계산서 등록 · 견적서 발행 · 업체관리 · 수리부품 관리 · 브랜드&툴 관리
- **기준정보 관리 (`/master/`):**
    - 매출처 · 의뢰업체 · 단가 그룹 · 브랜드&툴 관리
- **장비/툴 관리 (`/inventory/`):**
    - 대시보드 · 통합 입출고 이력 · 입고 등록 · 출고 등록 · 재고 · 기준 정보 연결
- **시스템 관리 (`/sysadmin/`):**
    - 사용자(회원) 관리 · 전체 활동 기록
- **근무/근태 관리 (`/hr/`):**
    - 내 근무 달력 · 근태 기록 목록
