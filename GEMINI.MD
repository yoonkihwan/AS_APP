# 사내용 AS 시스템 개발 명세서

## 1. 프로젝트 개요

- **프로젝트명:** 사내용 AS 관리 시스템 (Internal AS System)
- **목표:** AS 입고부터 수리, 출고, 이력 관리까지의 전 과정을 전산화하여 업무 효율성 증대 및 데이터 누락 방지.
- **핵심 가치:** 쉽고 빠른 입력(드롭다운, 일괄 입고), 정확한 이력 추적, 마스터 데이터 기반의 체계적 관리.

---

## 2. 기술 스택 (Tech Stack)

| 구분            | 기술              | 설명                                                            |
| :-------------- | :---------------- | :-------------------------------------------------------------- |
| **Language**    | **Python**        | 전체 비즈니스 로직 처리 및 백엔드 구현                          |

## 3. 기능 구현 현황 (Implementation Status)

### 3.1. 마스터 데이터 관리 ✅ 구현 완료
- **매출처 (Company):** 업체명, 사업자등록번호, 대표자, 주소, 단가 그룹 관리.
- **단가 그룹 (CompanyCategory):** 업체별 단가 적용을 위한 그룹 관리 (사이드바 숨김 — Company Admin에서 통합 관리).
- **브랜드 (Brand) & 툴 (Tool):** 브랜드별 장비 모델 관리. 브랜드는 사이드바 숨김 — Tool Admin에서 autocomplete 검색용.
- **부품 (Part):** 부품명, 코드, 단가, 구분(`part_type`: 부품/공임) 관리. `filter_horizontal`로 여러 툴에 적용 가능. 부품 목록에서 구분 필터를 통해 부품과 공임을 분류 조회 가능.
- **수리 세트 (RepairPreset):** 자주 사용하는 부품 조합을 프리셋으로 관리. 부품 관리 페이지에서 바로 접근 가능 (사이드바 숨김 — Part Admin에서 통합 관리).
- **의뢰업체 (OutsourceCompany):** 외주 수리 업체 관리 (사이드바 숨김 — 업체관리 탭에서 통합 접근).

### 3.2. 업체관리 ✅ 구현 완료
- 사이드바의 **"업체관리"** 메뉴 1개로 매출처업체와 의뢰업체를 통합 관리.
- 업체관리 페이지 진입 시 **탭 네비게이션**으로 매출처업체 / 의뢰업체 전환.

### 3.3. AS 프로세스 ✅ 구현 완료
- **일괄 입고 (InboundBatch):**
    - 날짜, 업체, 담당자 등 공통 정보를 1번 입력.
    - `TabularInline`을 통해 여러 장비(시리얼 포함)를 한 번에 등록.
    - 브랜드 선택 → 해당 브랜드 장비만 필터링 (Cascading Dropdown).
    - 쉼표(`,`)로 구분된 시리얼 번호 입력 시 자동으로 개별 티켓 생성.
    - 동일 배치 내 중복 검증 + 기존 활성 티켓 중복 검증.
- **AS 티켓 (ASTicket):**
    - 입고~출고 전 과정을 하나의 티켓으로 관리.
    - 상태: 입고(Inbound) → 수리대기(Waiting) → 수리완료(Repaired) / 수리의뢰(Outsourced) / 자체폐기(Disposed) → 출고(Shipped).
- **수리 기록 (RepairTicket):**
    - 입고/수리대기/수리의뢰 상태인 티켓만 필터링하여 표시.
    - 해당 장비에 적용 가능한 부품만 체크박스 위젯으로 표시 (부품/공임 섹션 시각적 분리).
    - 부품/공임(M2M) 선택 시 수리 비용 자동 계산 + 수리완료 상태 자동 변경.
    - **수리 세트 프리셋 (다중 선택):** 여러 수리 세트를 토글 방식으로 동시 적용 가능. 부품은 누적 선택되며, **공임은 선택된 세트들 중 최고가 공임 1개만 자동 선택** (중복 과금 방지).
    - 부품 목록 API (`api/parts-for-tool/<tool_id>/`) 제공.
- **출고 처리 (OutboundTicket):**
    - 수리 완료된 티켓만 필터링.
    - 목록에서 체크박스 선택 후 '출고 처리' 액션으로 일괄 상태 변경.
    - 오늘 날짜 출고 / 날짜 선택 출고 두 가지 액션 제공.
    - 견적서 추출 여부, 세금계산서 발행 여부 체크.
- **통합 이력 (ASHistory):**
    - 전체 이력 조회 전용 (Read-Only) 및 데모용 견적서 PDF 파일 렌더링.
    - **상태 정정 액션:** 체크박스 선택 후 상태 변경 가능 (혼합 선택 시 에러 메시지 표시).
        - 출고 취소 → 수리완료로 되돌리기 (출고일 초기화)
        - 수리완료 취소 → 입고로 되돌리기 (사용 부품/공임 초기화, 수리 비용 0원)
        - 자체폐기 취소 → 입고로 되돌리기
        - 견적서 추출 상태 초기화
        - 세금계산서 발행 상태 초기화
        - 입고 → 자체폐기 처리
- **견적서 발행 (데모) (추출 메뉴):**
    - AS 관리와 기준정보 사이 새로운 그룹인 '추출'로 분리.
    - 수리완료 또는 출고 상태인 데이터들을 다스 공구실 견적서 엑셀 템플릿 양식과 완벽히 동일한 구조의 PDF 파일로 생성/다운로드 제공.
- **세금계산서 등록 (AS 관리 메뉴):**
    - 출고 등록 하단에 배치.
    - 출고된 데이터를 조회하여 체크박스를 통해 간단히 '세금계산서 발행' 여부 마킹 (내부 확인용).
    - 간결한 화면 구성을 위해 장비/툴과 시리얼 번호를 분리 (고장 증상 열 제거).
    - 다양한 필터(업체, 날짜, 상태, 브랜드 등) 및 검색 제공.
    - **상태별 행 색상 적용:** 목록 화면에서 뱃지 대신 행(Row) 전체의 배경색을 변경하여 입고(노란색), 수리완료(초록색), 출고(파란색), 자체폐기(빨간색) 상태를 훨씬 빠르고 직관적으로 구별.

### 3.4. 대시보드 ✅ 구현 완료
- **KPI 카드:** 수리 대기 · 이번 달 입고 · 수리 완료(미출고) · 이번 달 출고.
- **수리 대기 목록:** 최근 10건 표시 (장비/시리얼 포함).
- **최근 출고 목록:** 최근 10건 표시.
- **바로가기 버튼:** 수리 기록 · 출고 등록 · 통합 이력 페이지 빠른 이동.

### 3.5. 장비/툴 입출고 관리 (Tool Inventory) ✅ 구현 완료
- **초점:** 기존 파이썬 데스크탑 형태의 재고 관리 프로그램을 웹으로 이관.
- **기능:**
    - AS 시스템과 사이트 분리 (`/inventory/`)
    - **공유 데이터:** `Brand`, `Tool` 마스터 데이터는 AS 앱과 공유하여 Single Source of Truth 달성.
    - **통합 입출고 이력:** 기존 단일 데이터베이스(재고 리스트)를 읽기 전용으로 시각화하여 제공하며 등록 버튼을 비활성화 함.
    - **입고 등록:** 배치(Batch) 단위 등록 지원, 시리얼 번호 다중 입력시 자동 분리 등록 기능.
    - **출고 등록 (인라인 UI 도입):** 
        - 1회 출고 시 여러 장비를 인라인 폼(TabularInline)으로 한 번에 등록.
        - 모델(Tool) 선택 시 비동기(API)로 해당 모델의 **현재 재고량** 실시간 박스 표기 및 **선택 가능한 재고목록(시리얼)**이 체크박스로 노출.
        - 미선택 시 단순 수량 설정 통한 선입선출 출고 로직 지원.
    - **재고 현황 출력 (ToolStockSummary):** 실시간 툴별 요약 재고와 시리얼 정보들을 제공하며, "재고 내역 출력" 버튼(글로벌 액션) 하나로 전체 데이터를 reportlab을 활용해 PDF로 깔끔하게 떨어트리는 기능을 지원합니다.

### 3.6. 시스템 계정 관리 (Sysadmin) ✅ 구현 완료
- **초점:** 직원 계정 생성 후 관리자가 권한을 쉽게 부여할 수 있도록 함.
- **기능:** 
    - 시스템 관리 사이트 분리 (`/sysadmin/`)
    - 관리자 목록 액션 탭에 **"선택 항목 일괄: 일반업무 권한 허용"** 액션 추가.
    - 개별 유저 수정 화면에서 **"일반 업무 권한 일괄 추가"** 체크박스를 통해 전체 권한 부여를 쉽게 처리.

### 3.7. 근무/근태 관리 (HR App) (개발 예정/진행 중) 🚀
- **초점:** 엑셀로 관리하던 직원들의 연장근무, 휴가, 특근 내역을 웹 상에서 쉽게 기록하고 조회 가능하도록 함.
- **기능:**
    - 근무/근태 관리 사이트 분리 (`/hr/`)
    - 달력(Calendar) 형태의 직관적인 UI 제공.
    - 월별 근무 데이터 조회 및 급여용 데이터(PDF/Excel) 추출 기능.
    - 근무 기록(`AttendanceRecord`) 모델 개발 및 Unfold Admin에 연결.

## 4. 데이터베이스 설계 (Schema Design)

### 4.1. 주요 모델 (마스터 데이터)
- **CompanyCategory:** 단가 그룹
- **Company:** 매출처 정보 (단가 그룹 FK 포함)
- **Brand:** 브랜드 정보 (AS 시스템과 Tool Inventory 공유)
- **Tool:** 장비 정보 (브랜드 FK, unique_together: brand + model_name) (공유)
- **Part:** 부품 정보 (Tool M2M - 다중 장비 호환, `part_type`: 부품/공임 구분)
- **RepairPreset:** 수리 세트 (Part M2M, Tool M2M)
- **OutsourceCompany:** 의뢰업체 정보 (업체명, 연락처, 메모)

### 4.2. 트랜잭션 모델
- **InboundBatch / ASTicket:** AS 통합 티켓 
- **Tool Inventory 모델 (tool_inventory app)**
    - **InventoryBatch:** 장비 입고 헤더 (입고일, 입고처, 담당자)
    - **Inventory:** 통함 입출고 이력 / 개별 단품(시리얼 등) 재고 상태 내역 (재고/출고)
    - **OutboundBatch:** 장비 출고 헤더 (출고일, 출고처)
    - **OutboundTicket:** 출고할 대상 품목 목록 (현재 M2M 제거되고 단순수량/시리얼 Char 로직으로 변경됨)
- **근무/근태 관리 모델 (hr_app)**
    - **AttendanceRecord:** 개별 직원의 일별 근무형태, 시간, 연장/특근 시간 등 기록

### 4.3. 프록시 모델 (Admin 뷰 분리)
| 프록시 모델       | 용도              | 표시 상태                     |
|:-----------------|:------------------|:-----------------------------|
| `InboundTicket`  | 입고 등록 전용     | 입고                          |
| `RepairTicket`   | 수리 기록 전용     | 입고/수리대기/수리의뢰         |
| `OutboundTicket` | 출고 처리 전용     | 수리완료                      |
| `ASHistory`      | 통합 이력 조회     | 전체 (읽기 전용)              |
| `EstimateTicket` | 견적서 발행 (데모)  | 수리완료/출고 (양식동일 PDF 추출) |
| `TaxInvoiceTicket`| 세금계산서 등록     | 출고 (내부 확인용)              |
| `ToolStockSummary`| 재고 현황 요약     | Tool Inventory 전용, 읽기/PDF |

## 5. UI/UX 가이드라인 (Django Unfold)

현재 시스템은 **Django Unfold** 테마를 기반으로 하여, 직관적이고 모던한 Admin UI를 제공합니다.

### 5.1. 일반 원칙
- **Clean & Minimal:** 불필요한 장식 요소를 배제하고 데이터 가독성을 최우선으로 합니다. 다크 모드가 기본 테마입니다.
- **No Popup:** 팝업 창을 띄우는 대신, 현재 페이지 내에서 인라인 편집이나 자동완성을 활용합니다.
- **No Related Buttons:** `NoRelatedButtonsMixin`으로 FK/M2M 필드 옆 추가/수정/삭제 버튼을 제거하여 입력 실수 방지.
- **Keyboard Friendly:** 탭(Tab) 키 이동과 엔터(Enter) 키 제출이 자연스럽게 이어지도록 필드 순서를 배치합니다.
- **네비게이션 직관성 (`.cursorrules`):** 사이드바 메뉴명과 화면명 또한 완벽히 동기화.
- **시각적 구별성 (`.cursorrules`):** 티켓 상태 표시(입고-노란색, 등)에 **행 단위 배경색(Row Colors)** 을 도입하여 분류 식별성 극대화.
- **로컬라이징 완벽 지원 (`.cursorrules`):** '검색', '필터' 등 플레이스홀더를 사용자에 친화적인 한국어로 교체.

### 5.2. 입력 화면 (Forms)
- **자동완성 (Autocomplete):**
    - 업체(Company), 장비(Tool) 등 데이터가 많은 필드는 반드시 `autocomplete_fields`를 사용.
    - 브라우저 기본 자동완성(`autocomplete="off"`)은 비활성화.
- **인라인 편집 (Inlines):**
    - 입고/출고 처리 시 `TabularInline`을 사용하여 빠르게 여러 행을 한 화면에 다중 처리.
- **동적 필드 제어 (Vanilla JS):**
    - 출고 폼과 같이 브랜드 선택 -> 툴 목록 갱신 -> 재고 목록 및 수량 뷰어 갱신 등의 실시간 통신은 순수 JavaScript `fetch` API 기반으로 구성. CheckboxSelectMultiple을 통해 다중 선택 UI 제공.
- **체크박스 위젯:**
    - 수리 기록 부품 선택 및 출고 대상 시리얼 식별 모두 직관적인 Checkbox 리스트로 전개.
- **필드 그룹화 (Fieldsets):**
    - 관련 있는 필드끼리 묶어서 표시 (예: '입고 정보', '수리 부품 선택').
- **저장 후 동작:**
    - 불필요한 부가 저장 버튼(저장 후 계속 편집, 저장 후 새로 추가 등)은 CSS로 일괄 숨김 처리하여 사용자가 "저장" 버튼만 클릭하도록 최소화.

### 5.3. 목록 화면 (ChangeList)
- **일괄 액션 (Actions):** 체크박스 선택 후 일괄 출고 처리 혹은 일괄 권한 부여 등의 효율적인 액션 활용.
- **탭 네비게이션:** 업체관리 페이지 등 상단 탭을 통해 데이터 성격을 분류.

### 5.4. 다중 사이트 네비게이션 (Navigation)
하나의 프로젝트에 **AS 관리**, **툴 입출고 관리**, **시스템 계정 관리**를 각각 독립적인 하위 사이트(AdminSite)로 분리운영.
- **AS 관리 (`/admin/`):** 
    - 통합 이력 · 입고 등록 · 수리 기록 등록 · 출고 등록 등 메인 액션 
- **장비/툴 관리 (`/inventory/`):**
    - 통합 재고 리스트 · 입고 등록 · 출고 처리.
- **시스템 관리 (`/sysadmin/`):**
    - 사용자(회원) 관리 및 권한 그룹.
- **근무/근태 관리 (`/hr/`):**
    - 본인 또는 전 직원의 근무 기록(연장근무, 휴가 등) 달력 UI 제공 및 급여용 정산 자료 출력.

## 6. 향후 개선 계획 (Roadmap)

- **견적서 출력:** 기존 엑셀 폼(다스 공구실 양식)과 완전히 동일한 레이아웃 표 서식을 ReportLab을 이용해 PDF 원클릭 다운로드 기능 연동 완료.
- **단가 그룹별 차등 가격:** 업체별 단가 그룹에 따른 가격 자동 적용.
- **분해도 이미지 매핑:** 이미지 클릭으로 부품 정보 연동.
- **모바일 대응:** 현장에서 모바일 기기로 간단한 상태 변경 가능하도록 반응형 최적화.
