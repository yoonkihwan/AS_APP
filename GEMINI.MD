# 사내용 AS 시스템 개발 명세서

## 1. 프로젝트 개요

- **프로젝트명:** 사내용 AS 관리 시스템 (Internal AS System)
- **목표:** AS 입고부터 수리, 출고, 이력 관리까지의 전 과정을 전산화하여 업무 효율성 증대 및 데이터 누락 방지.
- **핵심 가치:** 쉽고 빠른 입력(드롭다운, 일괄 입고), 정확한 이력 추적, 마스터 데이터 기반의 체계적 관리.

---

## 2. 기술 스택 (Tech Stack)

| 구분            | 기술              | 설명                                                            |
| :-------------- | :---------------- | :-------------------------------------------------------------- |
| **Language**    | **Python**        | 전체 비즈니스 로직 처리 및 백엔드 구현                          |
| **Framework**   | **Django 5.x**    | 웹 애플리케이션 프레임워크 (MVT 패턴, ORM 활용)                 |
| **Database**    | **SQLite** (로컬) / **PostgreSQL** (운영) | 로컬 개발용 SQLite, Render 배포용 PostgreSQL |
| **Frontend/UI** | **Django Unfold** | Django Admin을 현대적인 UI로 커스터마이징하여 대시보드 구축     |
| **Import/Export** | **django-import-export** | 데이터 일괄 가져오기/내보내기 지원                        |
| **Hosting**     | **Render**        | 운영 환경 호스팅 (WhiteNoise 정적 파일 서빙)                   |

## 3. 기능 구현 현황 (Implementation Status)

### 3.1. 마스터 데이터 관리 ✅ 구현 완료
- **매출처 (Company):** 업체명, 지역/국가, 주소, 단가 그룹 관리.
- **단가 그룹 (CompanyCategory):** 업체별 단가 적용을 위한 그룹 관리 (사이드바 숨김 — Company Admin에서 통합 관리).
- **브랜드 (Brand) & 툴 (Tool):** 브랜드별 장비 모델 관리. 브랜드는 사이드바 숨김 — Tool Admin에서 autocomplete 검색용.
- **부품 (Part):** 부품명, 코드, 단가, 구분(전용/공용/공임) 관리. `filter_horizontal`로 여러 툴에 적용 가능.
- **수리 세트 (RepairPreset):** 자주 사용하는 부품 조합을 프리셋으로 관리. 부품 관리 페이지에서 바로 접근 가능 (사이드바 숨김 — Part Admin에서 통합 관리).
- **의뢰업체 (OutsourceCompany):** 외주 수리 업체 관리 (사이드바 숨김 — 업체관리 탭에서 통합 접근).

### 3.2. 업체관리 ✅ 구현 완료
- 사이드바의 **"업체관리"** 메뉴 1개로 매출처업체와 의뢰업체를 통합 관리.
- 업체관리 페이지 진입 시 **탭 네비게이션**으로 매출처업체 / 의뢰업체 전환.

### 3.3. AS 프로세스 ✅ 구현 완료
- **일괄 입고 (InboundBatch):**
    - 날짜, 업체, 담당자 등 공통 정보를 1번 입력.
    - `TabularInline`을 통해 여러 장비(시리얼 포함)를 한 번에 등록.
    - 브랜드 선택 → 해당 브랜드 장비만 필터링 (Cascading Dropdown).
    - 쉼표(`,`)로 구분된 시리얼 번호 입력 시 자동으로 개별 티켓 생성.
    - 동일 배치 내 중복 검증 + 기존 활성 티켓 중복 검증.
- **AS 티켓 (ASTicket):**
    - 입고~출고 전 과정을 하나의 티켓으로 관리.
    - 상태: 입고(Inbound) → 수리대기(Waiting) → 수리완료(Repaired) / 수리의뢰(Outsourced) / 자체폐기(Disposed) → 출고(Shipped).
- **수리 기록 (RepairTicket):**
    - 입고/수리대기/수리의뢰 상태인 티켓만 필터링하여 표시.
    - 해당 장비에 적용 가능한 부품만 체크박스 위젯으로 표시.
    - 부품/공임(M2M) 선택 시 수리 비용 자동 계산 + 수리완료 상태 자동 변경.
    - **수리 세트 프리셋:** 자주 사용하는 부품 조합을 원클릭으로 적용 가능.
    - 부품 목록 API (`api/parts-for-tool/<tool_id>/`) 제공.
- **출고 처리 (OutboundTicket):**
    - 수리 완료된 티켓만 필터링.
    - 목록에서 체크박스 선택 후 '출고 처리' 액션으로 일괄 상태 변경.
    - 오늘 날짜 출고 / 날짜 선택 출고 두 가지 액션 제공.
    - 견적서 추출 여부, 세금계산서 발행 여부 체크.
- **통합 이력 (ASHistory):**
    - 전체 이력 조회 전용 (Read-Only).
    - 다양한 필터(업체, 날짜, 상태, 브랜드 등) 및 검색 제공.
    - date_hierarchy로 날짜별 탐색 가능.
    - **상태별 컬러 뱃지:** 입고(노란색), 수리완료(초록색), 출고(파란색), 자체폐기(빨간색)로 한눈에 구분.

### 3.4. 대시보드 ✅ 구현 완료
- **KPI 카드:** 수리 대기 · 이번 달 입고 · 수리 완료(미출고) · 이번 달 출고.
- **수리 대기 목록:** 최근 10건 표시 (장비/시리얼 포함).
- **최근 출고 목록:** 최근 10건 표시.
- **바로가기 버튼:** 수리 기록 · 출고 등록 · 통합 이력 페이지 빠른 이동.

## 4. 데이터베이스 설계 (Schema Design)

### 4.1. 주요 모델 (마스터 데이터)
- **CompanyCategory:** 단가 그룹
- **Company:** 매출처 정보 (단가 그룹 FK 포함)
- **Brand:** 브랜드 정보
- **Tool:** 장비 정보 (브랜드 FK, unique_together: brand + model_name)
- **Part:** 부품 정보 (Tool M2M - 다중 장비 호환, 구분: 전용/공용/공임)
- **RepairPreset:** 수리 세트 (Part M2M, Tool M2M)
- **OutsourceCompany:** 의뢰업체 정보 (업체명, 연락처, 메모)

### 4.2. 트랜잭션 모델
- **InboundBatch:** 일괄 입고 헤더 (날짜, 업체, 담당자, 메모)
- **ASTicket:** AS 통합 티켓
    - FK: InboundBatch, Company, Tool, OutsourceCompany
    - M2M: used_parts (사용 부품/공임)
    - Status: 입고/수리대기/수리의뢰/수리완료/출고/자체폐기
    - Dates: 입고일, 출고일
    - Flags: 견적서 추출, 세금계산서 발행
    - Auto: repair_cost (사용 부품 합산 자동 계산)

### 4.3. 프록시 모델 (Admin 뷰 분리)
| 프록시 모델       | 용도              | 표시 상태                     |
|:-----------------|:------------------|:-----------------------------|
| `InboundTicket`  | 입고 등록 전용     | 입고                          |
| `RepairTicket`   | 수리 기록 전용     | 입고/수리대기/수리의뢰         |
| `OutboundTicket` | 출고 처리 전용     | 수리완료                      |
| `ASHistory`      | 통합 이력 조회     | 전체 (읽기 전용)              |

## 5. UI/UX 가이드라인 (Django Unfold)

현재 시스템은 **Django Unfold** 테마를 기반으로 하여, 직관적이고 모던한 Admin UI를 제공합니다.

### 5.1. 일반 원칙
- **Clean & Minimal:** 불필요한 장식 요소를 배제하고 데이터 가독성을 최우선으로 합니다.
- **No Popup:** 팝업 창을 띄우는 대신, 현재 페이지 내에서 인라인 편집이나 자동완성을 활용합니다.
- **No Related Buttons:** `NoRelatedButtonsMixin`으로 FK/M2M 필드 옆 추가/수정/삭제 버튼을 제거하여 입력 실수 방지.
- **Keyboard Friendly:** 탭(Tab) 키 이동과 엔터(Enter) 키 제출이 자연스럽게 이어지도록 필드 순서를 배치합니다.
- **네비게이션 직관성 (`.cursorrules`):** 시스템 전체의 루트 및 브레드크럼(Breadcrumb) 링크를 "대시보드"로 통일하여 예측 가능한 이동 동선을 설계했습니다. 사이드바 메뉴명과 화면명 또한 완벽히 동기화.
- **시각적 구별성 (`.cursorrules`):** 티켓 상태 표시(입고-노란색, 수리완료-초록색, 출고-파란색, 자체폐기-빨간색)에 컬러 뱃지를 도입하여 가독성을 극대화했습니다.
- **로컬라이징 완벽 지원 (`.cursorrules`):** 영어로 하드코딩된 'Search', 'Filters' 등의 플레이스홀더와 버튼들을 모두 사용자에 친화적인 한국어 ('검색', '필터')로 교체했습니다.

### 5.2. 입력 화면 (Forms)
- **자동완성 (Autocomplete):**
    - 업체(Company), 장비(Tool) 등 데이터가 많은 필드는 반드시 `autocomplete_fields`를 사용.
    - 브라우저 기본 자동완성(`autocomplete="off"`)은 비활성화.
- **인라인 편집 (Inlines):**
    - 입고 등록 시 `TabularInline`을 사용하여 빠르게 여러 행을 입력.
    - 불필요한 추가/수정/삭제 아이콘(`NoRelatedButtonsMixin`)을 제거.
- **체크박스 위젯:**
    - 수리 기록의 부품 선택은 `CheckboxSelectMultiple` 위젯으로 직관적 선택.
- **필드 그룹화 (Fieldsets):**
    - 관련 있는 필드끼리 묶어서 표시 (예: '입고 정보', '수리 부품 선택').
- **저장 후 동작:**
    - 입고: 저장 후 새 입고 등록 폼으로 자동 이동.
    - 수리: 저장 후 수리 기록 목록으로 자동 이동.
    - 부가 저장 버튼(저장 후 계속 편집, 저장 후 새로 추가) 숨김 처리.

### 5.3. 목록 화면 (ChangeList)
- **필터링 (Filters):** 우측 사이드바 필터 — 상태, 업체, 날짜별 좁혀보기.
- **검색 (Search):** 시리얼 번호, 업체명, 모델명, 브랜드명으로 통합 검색.
- **일괄 액션 (Actions):** 체크박스 선택 후 일괄 출고 처리 (오늘/날짜 선택).
- **커스텀 버튼:** 수리 기록 목록에서 행별 "🔧 수리 기록 등록" 버튼으로 해당 티켓 편집 이동.
- **탭 네비게이션:** 업체관리 페이지에서 매출처업체/의뢰업체 탭 전환.

### 5.4. 네비게이션 (Navigation)
- **사이드바 구성:**
    - **대시보드:** KPI + 요약 정보.
    - **AS 관리:** 통합 이력 · 입고 등록 · 수리 기록 등록 · 출고 등록 (업무 흐름순).
    - **기준 정보:** 업체관리(탭: 매출처업체/의뢰업체) · 수리부품 관리 · 브랜드 & 툴 관리.
    - 사용하지 않는 메뉴는 사이드바에서 숨김 처리 (`has_module_permission = False`).

## 6. 향후 개선 계획 (Roadmap)

- **견적서 출력:** 엑셀 폼 연동하여 견적서 파일 생성 및 다운로드 기능.
- **단가 그룹별 차등 가격:** 업체별 단가 그룹에 따른 가격 자동 적용.
- **분해도 이미지 매핑:** 이미지 클릭으로 부품 정보 연동.
- **모바일 대응:** 현장에서 모바일 기기로 간단한 상태 변경 가능하도록 반응형 최적화.
