# TOOL_INVENTORY 개발 명세서 (기존 프로그램 웹 전환)

## 1. 개요
*   **시스템 명:** 장비/툴 입출고 관리 시스템 (Tool & Inventory Management)
*   **목적:** 기존 데스크톱 파이썬 프로그램(`TOOL 입출고 관리.exe`)과 `inventory.db`의 데이터를 웹 애플리케이션으로 전환하여 AS 프로젝트와 같은 서버에서 작동하도록 함.
*   **주요 특징:** 
    * AS 시스템(`as_app`)과는 데이터 통신은 최소화하되, **기준 정보(Brand, Tool)는 공유(Single Source of Truth)**.
    * 화면과 접근 경로는 완전히 격리하여 운영 (다중 Admin Site 구조).
    * 대량의 인라인 UI 처리를 순수 Javascript와 연동 및 실시간 재고 PDF 다운로드 지원.

---

## 2. 기존 데이터베이스 분석 및 테이블 매핑
기존 프로그램(`inventory.db`)은 매우 직관적인 텍스트 기반의 데이터 구조를 가지고 있습니다. 이것을 장고의 관계형 데이터베이스(PostgreSQL) 형태로 변환하여 구현합니다.

### 2.1 마스터 테이블
기존 데이터베이스엔 단순히 이름만 관리되는 목록들이 있습니다. 이번 프로젝트로 전환하며 **AS 앱(`as_app.models`)의 기준 정보를 사용**합니다.
*   `Brand` (as_app) - 브랜드
*   `Tool` (as_app) - 브랜드별 장비/툴

### 2.2 메인 테이블 (입출고 내역)
*   **Inventory (통합 입출고 이력):** 개별 장비(시리얼 단위) 재고 테이블. 단순 조회용도로만 관리자 뷰에 공개되며 `+` 추가 버튼 등은 비활성화됩니다.
    *   `id`: `UUID` 형태의 기존 고유값. 장고에서도 `UUIDField(primary_key=True)`를 사용하여 기존 ID 유지.
    *   `batch`: 입고 배치 (새로운 연동 구조)
    *   `tool`: AS 앱의 Tool 모델 연동 (Foreign Key)
    *   `serial`: 시리얼 번호 (`CharField`, 선택 입력 가능)

*   **InboundBatch (입고 테이블):** 다수 입력 처리를 위한 래퍼(Wrapper) 
    * 날짜, 공급사, 담당자 헤더 정보 기록.
    * 인라인 모델을 통해 `Inventory`를 다건 생성하고, `,`로 구분된 시리얼 번호를 자동 스플릿.

*   **OutboundBatch (출고 테이블):** 다건 출고 처리를 위한 헤더
    * 날짜, 매출처 헤더 정보 기록.
    * 인라인을 통해 다수 `OutboundTicket` 생성.

*   **OutboundTicket (출고 티켓 데이터):** 특정 툴을 몇 개, 혹은 특정 시리얼을 지정하여 출고
    * 화면에서 `Brand` 선택 시 `Tool` 목록이 비동기로 갱신됨.
    * `Tool` 선택 시 해당 툴에 해당하는 재고 목록(시리얼유/무)이 브라우저 하단에 동적 체크박스로 표시, 수량이 계산됨(FIFO 지원).

### 2.3 프록시 모델 (조회/출력 전용)
*   **ToolStockSummary (재고 현황):** `Tool` 마스터 모델을 상속받아, 현재 `Inventory` 테이블 기준 상태가 '재고'인 데이터를 툴 종류별로 그룹핑합니다.
    * 툴별 **보유 수량** 및 **시리얼 목록** 집계를 한 눈에 확인 가능.
    * 좌측 체크박스 없이 상단에 "재고 내역 출력 (PDF)" 글로벌 버튼이 위치하여 전체 재고 현황을 인쇄용 PDF 파일로 다운로드 받을 수 있습니다.

## 3. 화면 구성 및 분리 원칙 (Admin Site 분리)
AS 관리와 툴 인벤토리 관리를 하나의 서버에서 운영하되, 사용자가 작업 도중 헷갈리지 않게 **접속 경로와 관리자 화면 전체를 분리**합니다.

*   `as.yoursite.com/inventory/`: **툴/장비 관리 시스템** (본 앱 관련 기능만 노출)

본 앱(Tool Inventory)의 Admin 화면은 다음을 충족합니다:
*   **대시보드:** 별도의 독립적인 대시보드를 구성.
*   **메인 관리 화면:**
    *   "통합 입출고 이력 (기존 인벤토리)", "입고 등록", "출고 등록", "재고" 절차 별로 메뉴 분리.
    *   다건 배치를 위해 API 라우팅과 `JS` 파일 (`outbound_form.js`)을 기반으로 빠르고 정확한 조작 확보.

## 4. 디자인 제약사항 (Django Unfold)
- "저장 후 계속", "다른 이름으로 저장" 등의 **불필요한 버튼은 모두 CSS를 사용해 강제 마스킹 (`outbound_form.css`)** 처리, 혼선을 방지.
- PDF 출력 등 리스트 전체를 포괄하는 기능은 `actions_list`로 글로벌 버튼화 하여 직관적인 출력이 가능하게 합니다.
