# 견적서 작성 가이드 (엑셀 템플릿 기반)

이 문서는 `견적서/다스 공구실 락볼트 수리비 견적.xlsx` 파일의 구조를 분석하여, 향후 시스템에서 자동으로 견적서를 생성(추출)할 때 참고할 수 있도록 작성된 가이드입니다. 

## 1. 기본 정보 및 시트 구조
- **문서 양식**: 엑셀 (xlsx)
- **템플릿 용도**: 고객사(예: ㈜다스)에 제출하는 툴 수리비(Lock Bolt 등) 견적서
- **데이터 매핑 구조**: 엑셀의 특정 셀(행, 열)에 견적서의 헤더(수신자, 날짜 등) 및 본문(품목 리스트, 합계) 정보가 고정적으로 위치해 있습니다.

## 2. 셀 매핑 (Cell Mapping) 가이드

아래는 1-based index (엑셀의 기본 행 번호) 및 Column (A, B, C...) 기준에 따른 데이터 삽입 위치입니다.

### [1] 헤더(공통) 정보 영역
| 항목명 | 엑셀 행 번호 | 엑셀 열 번호 | 설명 |
| :--- | :--- | :--- | :--- |
| **수신자 (고객사)** | 4행 | A열 | 예: `㈜다스 귀중` (B열에 "귀중" 고정될 수 있음) |
| **참고사항** | 5행 | A열 | `참    조 : ` 이후에 참고인 기입 |
| **총 금액(한글 등)** | 6행 | A열 | `금    액 : ` 이후에 총액 기입 |
| **납품기일** | 7행 | C열 | `납품기일 : 발주후 [  ] 일` (C열에 일수 기입) |
| **유효기간** | 9행 | C열 | `유효기간 : [  ] 일` (C열에 일수 기입) |
| **견적일자** | 10행 | A열/C열 | `견적일자 : YYYY 년 MM 월 [ ] 일` |
| **견적서 제목** | 11행 | A열 | 예: `제    목 : 락볼트 수리비 견적` |

### [2] 공급자(발신자) 정보 영역 (보통 템플릿에 하드코딩 되어 있음)
- **주소**: 6행 E열 (`주소 : 울산광역시 북구 호수2로 7-1(호계동)`)
- **연락처**: 7행 E열 (`Tel. (052) 988-0776 fax. 988-0775`)
- **모바일**: 8행 E열 (`M. 010-2577-0776`)
- **이메일**: 9행 E열 (`Mail. tooleng@naver.com`)

### [3] 품목(부품/공임) 리스트 영역
견적서의 본문으로 품목 리스트가 나열되며, 14행부터 헤더가 시작되고 16행부터 실제 데이터가 들어갑니다.

**테이블 헤더 (14행)**
- A14: `Description` (품명/내역)
- B14: `Model` (모델명/품번)
- E14: `Qty` (수량)
- F14: `Unit price` (단가)
- G14: `Amount` (금액)
- H14: `Remark` (비고)

*참고: 15행(A15)에는 대표 수리 대상 툴의 명칭 및 시리얼 번호가 기입될 수 있습니다. (예: `Lock Bolt 본체 수리비(Ser.No 868/2022)`)*

**데이터 기입 (16행 ~ )**
각 행마다 위의 열 구조에 맞추어 품목을 나열합니다. 
> *예시 (17행)*
> - A17: `Pull Intensifier Spring`
> - B17: `07267-08008`
> - E17: `1`
> - F17: `84000`
> - G17: `84000`
> - H17: `O-Ring 포함`

### [4] 금액 합계 및 하단 영역
품목 리스트가 끝난 직후 하단에 위치합니다. (데이터 건수에 따라 동적으로 행 번호 유지/변경 필요)
*아래 예시는 데이터가 19행에서 끝났을 때 기준입니다.*

- **공급가액 합계 (TOTAL)** 
  - F행(예: F20): `TOTAL`
  - G행(예: G20): 총 합계 금액 (예: `707000`)
- **최종네고가 (할인액 반영 시)**
  - F행(예: F22): `최종네고가`
  - G행(예: G22): 네고된 최종 금액 (예: `636000`)
- **VAT (부가세)**
  - F행(예: F23): `- VAT 별도 -` 또는 부가세 포함액 산정 안내
- **특기사항**
  - A행(예: A24): `특기사항 : ` 하단에 추가 내용 기입

## 3. 데모 데이터 추출 및 생성 로직 설계
향후 파이썬 등(예: `openpyxl`)을 이용해 이 엑셀 폼에 데이터를 채워넣기 위해 다음 플로우를 권장합니다.
1. **템플릿 로드**: `openpyxl.load_workbook()`으로 기존 견적서 `.xlsx` 양식을 읽어옴.
2. **헤더 데이터 채우기**: A4~A11 등의 고정된 셀 좌표에 클라이언트, 견적일자, 납기일, 프로젝트명(제목)을 주입.
3. **품목 리스트 채우기 (동적 확장)**:
   - 16행부터 시작하여 수리 부품(Parts) 개수만큼 데이터 입력.
   - 만약 템플릿의 여분 행(Row)을 초과할 경우 `insert_rows()`로 행을 추가하여 서식을 복사.
4. **합계 계산식 삽입/업데이트**: TOTAL, 최종네고가 셀의 값을 파이썬에서 합산하여 삽입하거나, 엑셀 SUM() 수식으로 반영.
5. **저장**: 새로운 파일명(예: `[고객사명]_견적서_YYYYMMDD.xlsx`)으로 저장하여 다운로드/추출 데모 완성.
